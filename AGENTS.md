# AGENTS.md — ghostshell

## Quick Facts
- Go module `ghostshell`; CI uses Go 1.25, builds must be static (`CGO_ENABLED=0`).
- Two binaries: `cmd/ghostshell` (CLI) and `cmd/ghostshell-daemon` (root collector daemon).
- **Linux only** — PTYs, `/proc`, `SO_PEERCRED`, unix sockets, `chattr`. The test suite cannot run on Windows/macOS; use a Linux container (`golang:1.25`).
- Storage locations:
  - Local (fail-open fallback): `$GHOSTSHELL_DIR` (default `~/.local/share/ghostshell`).
  - Central (encrypted, root-only): `$GHOSTSHELL_CENTRAL_DIR` (default `/var/lib/ghostshell`, `root:root 0700`).
  - Encryption key at `<central>/.ghostshell.key` (`0600`, made immutable via `chattr +i`).

## Common Commands
- `make check` → the whole gate in order, fail-fast: gofmt → vet → staticcheck → golangci-lint → test → build. Mirrors CI.
- `make fmt` / `make vet` / `make test` / `make build`
- `make VERSION=x.y.z packages` → `nfpm` builds the RPM and DEB into `release/`
- `make install` → CLI to `/usr/bin`, daemon to `/usr/libexec/ghostshell-daemon`, systemd unit, completion, man page.

## CI / Release (GitHub Actions)
- **`.github/workflows/ci.yml`** — every push and PR: gofmt → `go vet` → staticcheck → golangci-lint → `go test` → `make build`. golangci-lint is pinned to `v1.64.8` and installed via `go install` so it is built with the same Go 1.25 toolchain (a prebuilt `latest` pulls v2, whose config schema is incompatible with `.golangci.yml`).
- **`.github/workflows/release.yml`** — triggered by a `vX.Y.Z` **tag**. Builds the static binaries + rpm + deb + `SHA256SUMS` and publishes a GitHub Release using the built-in `GITHUB_TOKEN` (no PAT needed). The version is derived from the tag.
- To cut a release: `git tag -a vX.Y.Z -m "…" && git push origin vX.Y.Z`.
  **Tag only after the code is merged** — the release is built from the tagged commit, so a stale tag ships stale artifacts.

## Gotchas
- Central casts are AES-256-GCM; read/write them only through the `store.OpenCast*` APIs.
- The daemon socket (`/run/ghostshell-daemon.sock`) is mode `0666`. Privacy is enforced by **file permissions on the store** plus **`SO_PEERCRED`** peer authentication — never by the socket mode. No command may trust a client-supplied identity.
- The daemon acknowledges a registered `REC` session with `OK\n`. The recorder waits for that ack and falls back to a user-local file on `ERR` / timeout / EOF. **Do not remove the ack** without also fixing the recorder — otherwise a rejected session is silently lost.
- `chattr +i` on the key **silently no-ops** on overlayfs, tmpfs and many network mounts. The daemon reads the flag back and warns loudly if it did not stick — never assume the key is immutable.
- The sshd `ForceCommand` wrapper is **opt-in** (`ghostshell init --enable-ssh-forcecommand`). The package deliberately does **not** install it.
- Tests that touch the store must point `GHOSTSHELL_DIR`, `GHOSTSHELL_CENTRAL_DIR` and `GHOSTSHELL_DAEMON_SOCK` at temporary locations.
- Losing `.ghostshell.key` makes every encrypted recording permanently unreadable.

## Ansible Integration
- Callback plugin `scripts/ansible/ghostshell.py` writes JSON Lines to `~/.local/share/ghostshell/ansible/<runid>.ajsonl` (`0600`), which the daemon ingests into the central store.
- The packages **do** install it, at `/usr/share/ghostshell/ansible/ghostshell.py`.
- Enable it with:
  ```
  ANSIBLE_CALLBACK_PLUGINS=/usr/share/ghostshell/ansible
  ANSIBLE_CALLBACKS_ENABLED=ghostshell
  ```

## Packaging Details
- `nfpm.yaml` builds the RPM and DEB, containing: the CLI, the daemon, the systemd unit, the man page, bash completion, the `profile.d` auto-record hook, the ssh wrapper, the Ansible callback plugin, the logrotate drop-in, and the default config.
- `make VERSION=x.y.z packages` exports `GHOSTSHELL_VERSION` for nfpm; artefacts land in `release/`.
- `SHA256SUMS` is generated by the release workflow and uploaded alongside the packages.
